# Angular UI Component

## Context
Follow Angular Foundation

## Goal
Create a presentational (dumb) UI component.

## Requirements
- No business logic
- Inputs fully typed
- Outputs explicit and typed
- No service injection unless strictly UI-related

## Constraints
- standalone + OnPush
- signals for local UI state if needed
- pure template

## Accessibility
- semantic HTML
- ARIA if needed

## Output
- component.ts
- component.html
- minimal styling with Tailwind
