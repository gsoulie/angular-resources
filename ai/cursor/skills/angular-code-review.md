# Angular Code Review

## Context
Follow Angular Foundation

## Goal
Review Angular code critically.

## Checkpoints

### Architecture
- smart vs dumb respected
- responsibilities clear

### Signals / RxJS
- signals used appropriately
- no unnecessary subscriptions

### Typing
- no any
- strict typing everywhere

### Templates
- no business logic
- no heavy expressions

### Performance
- OnPush used
- track in loops
- no unnecessary re-renders

### Accessibility
- semantic HTML
- ARIA correctness

### Maintainability
- readable
- not over-engineered

## Output
- issues list
- severity (low/medium/high)
- concrete fixes
